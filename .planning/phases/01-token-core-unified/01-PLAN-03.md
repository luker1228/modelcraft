<objective>
将 Gateway 端用户 GraphQL Handler 的 Token 验证从 HMAC-SHA256 切换到 ES256，与后端 Wave 2 的 token 签发方式保持一致。废弃 VerifyEndUserAccessToken 和 EndUserClaims，统一使用 VerifyAccessToken（ES256 公钥验证）。

**目的**：端用户 JWT 已改为 ES256 签发，Gateway 必须同步切换验证算法，否则所有端用户 GraphQL 请求均会返回 401。  
**输出**：4 个修改文件，Gateway 编译通过，端用户 GraphQL 端点可正常验证 ES256 token。  
**部署注意**：Backend 需先于 Gateway 部署（硬切策略）。
</objective>
