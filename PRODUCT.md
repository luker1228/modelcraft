# Product

## Register

product

## Users

企业内的业务技术负责人和运营配置人员。有基础技术认知（能理解"表结构""字段""角色权限"等概念），但不是后端工程师。主要场景：工作日在办公室的显示器前，处理配置任务，节奏紧凑，不希望为一个操作多想。核心需求是"快速找到想改的地方、改完、离开"——不需要被教育，不需要被鼓励，需要清晰的反馈和可预期的结构。

## Product Purpose

ModelCraft 是一个数据模型与权限配置平台。开发者/配置人员在此定义业务数据模型（表结构、字段、枚举）、配置数据库集群、管理 EndUser 权限体系（角色、权限包、权限点、行列策略），最终驱动 Runtime 查询的访问控制。

成功的样子：用户打开页面就知道该点哪里，配置过程没有意外，改完之后状态一目了然。

## Brand Personality

精准、可信、直接。Vercel Dashboard 是北极星参照：近乎单色的配色、大标题 + 小内容的层次对比、tab 下划线用前景色而非蓝色、表格扁平无背景、section 内容克制。

**已确定的具体决策：**
- Tab 导航：下划线式，激活态 `border-foreground`（黑色），不用蓝色
- Primary button：保持 `bg-primary`（蓝色），只用于创建/提交等主操作，不泛化
- Tab 内**不放子标题**（h2 + 描述文字）：Tab 本身已经标注，子标题是冗余
- 每个 Tab 内容区顶部只放 CTA 按钮（右对齐），不放标题区块

不炫耀，不哄人，把复杂的东西变得清楚。

参照对象：Vercel Dashboard（最高优先级）、Linear 的操作直觉、Stripe Dashboard 的表单严肃性。

## Anti-references

- Oracle / SAP 类企业软件的厚重 UI：过多层级、阴影叠阴影、按钮像砖头
- Canva / Shopify Admin 的消费化糖果感：圆润、彩色、鼓励性文案
- 炫技型玻璃拟态、渐变叠加、粒子背景——视觉噪音干扰操作判断
- 信息密度过低的"空气感"设计：大量留白但内容不知在哪
- 蓝色 accent 克制使用——只有创建/提交等主操作按钮和激活态（tab 下划线除外，激活 tab 用前景色）使用蓝色，列表、表格、说明性元素不出现蓝色

## Design Principles

1. **操作优先于展示。** 每个界面的首要任务是让用户完成一个动作，不是展示系统有多强大。
2. **层级明确，不藏东西。** 用户能一眼知道当前在哪个层级、有哪些可操作项、操作后会发生什么。
3. **密度服务于效率，呼吸感保障可读。** 合理的信息密度减少页面跳转，但密度不等于堆砌。行间距、分组间距、内边距要给内容留出呼吸空间——每一行都要赚到它存在的权利，同时不能因为挤压而让用户在视觉上迷失。
4. **状态可见，反馈即时。** Loading、空态、错误、成功，每种状态都有明确的视觉表达，不让用户猜。
5. **克制即风格，但层级要有重量。** Vercel 风格的克制不是平淡——是大标题配小内容、前景色 primary button、单色为主偶尔一点 accent。克制是手段，清晰层次是目的。

## Accessibility & Inclusion

暂无硬性 WCAG 要求。目标用户在标准办公环境下使用，优先保证操作清晰度和对比度可读性，避免纯色信息（不依赖颜色单独传达状态）。
