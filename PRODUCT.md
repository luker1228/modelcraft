# Product

## Register

product

## Users

企业内的业务技术负责人和运营配置人员。有基础技术认知（能理解"表结构""字段""角色权限"等概念），但不是后端工程师。主要场景：工作日在办公室的显示器前，处理配置任务，节奏紧凑，不希望为一个操作多想。核心需求是"快速找到想改的地方、改完、离开"——不需要被教育，不需要被鼓励，需要清晰的反馈和可预期的结构。

## Product Purpose

ModelCraft 是一个数据模型与权限配置平台。开发者/配置人员在此定义业务数据模型（表结构、字段、枚举）、配置数据库集群、管理 EndUser 权限体系（角色、权限包、权限点、行列策略），最终驱动 Runtime 查询的访问控制。

成功的样子：用户打开页面就知道该点哪里，配置过程没有意外，改完之后状态一目了然。

## Brand Personality

精准、可信、直接。**Stripe Dashboard 是北极星参照**：冷蓝灰的中性色系、shadow-first 的层级表达（而非 border-first）、压缩的标题尺寸、高密度表格、克制的单一强调色、功能性动效。

设计语言核心：**通过移除而非添加来创造高级感**。每一个元素都要赚到它存在的权利。

**已确定的具体决策：**
- 品牌强调色：`#4F46E5`（Indigo-600），仅用于主操作按钮、选中态、焦点环
- Tab 导航：下划线式，激活态 `border-primary`（Indigo），不用前景色
- Primary button：`bg-primary`（Indigo），只用于创建/提交等主操作，不泛化
- Tab 内**不放子标题**（h2 + 描述文字）：Tab 本身已经标注，子标题是冗余
- 每个 Tab 内容区顶部只放 CTA 按钮（右对齐），不放标题区块
- 字重上限：600（`font-semibold`），绝不使用 700/800/900
- 字体：Inter only，不使用 Space Grotesk
- 页面最大标题：20px/600，dashboard 场景不使用 32px 大标题
- 层级表达：shadow-first（极细多层阴影），卡片默认无显式 border 或仅 1px border
- 导航分组：section header 明确模块归属（11px uppercase muted），每组间有分割
- **页面背景色**：`#F6F8FA`（冷蓝灰，Stripe 标准）
- **表头（thead）**：纯白背景 + 2px 前景色下边框，无填充色，文字 `#1A1F36` uppercase
- **Toolbar 背景**：与页面背景同色 `#F6F8FA`；搜索框比 toolbar 深一档 `#EBEEF2`（border `#D8DDE5`），形成凹入质感

不炫耀，不哄人，把复杂的东西变得清楚。

## Anti-references

- Oracle / SAP 类企业软件的厚重 UI：过多层级、阴影叠阴影、按钮像砖头
- Canva / Shopify Admin 的消费化糖果感：圆润、彩色、鼓励性文案
- 炫技型玻璃拟态、渐变叠加、粒子背景——视觉噪音干扰操作判断
- 信息密度过低的"空气感"设计：大量留白但内容不知在哪
- 品牌色克制使用——只有创建/提交等主操作按钮和激活态使用 Indigo，列表、表格、说明性元素不出现品牌色
- 渐变 badge、药丸形（rounded-full）badge、彩色圆圈 empty state 装饰——模板感来源
- `hover:scale` 缩放动画、`backdrop-blur` 玻璃效果、半透明背景——消费级装饰
- 大面积品牌色块——dashboard 内部不做大色块

## Design Principles

1. **操作优先于展示。** 每个界面的首要任务是让用户完成一个动作，不是展示系统有多强大。
2. **层级明确，不藏东西。** 用户能一眼知道当前在哪个层级、有哪些可操作项、操作后会发生什么。
3. **密度服务于效率，呼吸感保障可读。** 合理的信息密度减少页面跳转，但密度不等于堆砌。行间距、分组间距、内边距要给内容留出呼吸空间。
4. **状态可见，反馈即时。** Loading、空态、错误、成功，每种状态都有明确的视觉表达，不让用户猜。
5. **克制即风格。** 通过移除而非添加创造高级感。字重最高 600，只有一个品牌色，卡片不堆装饰，动效只服务功能（150ms ease-out）。

## Accessibility & Inclusion

暂无硬性 WCAG 要求。目标用户在标准办公环境下使用，优先保证操作清晰度和对比度可读性，避免纯色信息（不依赖颜色单独传达状态）。
