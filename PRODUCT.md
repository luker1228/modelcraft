# Product

## Register

product

## Users

Primary users are enterprise data platform builders, backend engineers, and operations teams configuring data models, permissions, and runtime access in authenticated workspace flows. They usually work in admin contexts, switching between schema setup, access control, and verification tasks.

## Product Purpose

ModelCraft provides a single operational surface to define data structures and access policies with predictable, production-safe workflows. The product exists to reduce friction and mistakes in model configuration and authorization setup while keeping complex operations understandable.

## Brand Personality

Precise, dependable, restrained.
Tone should feel tool-like and professional: clear labels, explicit states, and low decorative noise so users can stay in task flow.

## Anti-references

Avoid consumer-social visual styles, playful gradients, and novelty-first interaction patterns. Avoid dashboard theatrics that prioritize visual effects over clarity (hero metrics, decorative glass, excessive motion).

## Design Principles

1. Task-first clarity: every layout decision should reduce cognitive load for active configuration work.
2. Familiar affordances over novelty: use standard product UI patterns users already trust.
3. State explicitness: loading, empty, error, selected, and success states must be visually unambiguous.
4. Controlled emphasis: use accent and hierarchy only for actions and current focus, never for decoration.
5. Consistent density: support data-dense workflows with predictable spacing and structure.

## Accessibility & Inclusion

Target WCAG AA contrast and keyboard-accessible interaction patterns on all primary controls. Keep motion subtle and optional by avoiding decorative animation and preferring quick state transitions. Use clear status text in addition to color for warnings/errors/success states.
