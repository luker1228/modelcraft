# 用户认证（登录 & 注册）

## 是什么问题

现有认证方案基于 Casdoor，但实际只用到登录和注册两个功能，Casdoor 的部署和维护成本远超实际需求。目标是用自研轻量方案替换 Casdoor，降低系统复杂度。

## 目标用户

国内开发者，使用手机号作为主要身份标识。

## 用户故事

- 作为新用户，我希望用手机号 + 密码注册账号，以便开始使用 ModelCraft。
- 作为已注册用户，我希望用手机号 + 密码登录，以便访问我的工作空间。

## 核心功能

**注册**
- 输入手机号 + 密码（+ 确认密码），提交后创建账号
- 手机号格式校验（11位国内号码）
- 手机号唯一性校验（已注册则提示）
- 密码强度要求：最少 8 位

**登录**
- 输入手机号 + 密码，验证通过后签发 JWT Token（有效期 7 天）
- 登录成功后跳转到 workspace 页（沿用现有逻辑）
- 错误提示：手机号不存在 / 密码错误

## 不做什么

- 不做短信验证码登录
- 不做第三方登录（微信、GitHub 等）
- 不做忘记密码 / 重置密码
- 不做邮箱注册
- 不做图形验证码 / 人机校验
- 不做多端 Token 管理 / 强制下线
- 不迁移 Casdoor 的其他能力（OAuth Server、权限管理等均不需要）
- 无存量用户数据，不需要数据迁移

## 替换 Casdoor 的影响范围

- 后端：移除 Casdoor SDK 依赖，自行实现 JWT 签发与校验
- 前端：替换登录/注册页面，调整 Token 存储与请求头携带方式
- 基础设施：Casdoor 服务可下线，减少一个部署依赖
