## 页面信息

- **页面名称**: 注册页
- **页面类型**: 编辑页（居中表单卡片）
- **路由路径**: /register
- **所属领域**: auth

## 数据依赖

| 领域实体 | 使用方式 | 关键字段 |
|----------|----------|----------|
| User (Aggregate Root) | 注册创建目标 | phoneNumber, password |
| PhoneNumber (Value Object) | 手机号输入与校验 | value（11 位国内手机号） |
| HashedPassword (Value Object) | 密码输入（前端提交明文，后端哈希存储） | 明文 → hash |

## 区块说明

### 1. 注册表单区

- **用途**: 收集新用户注册信息
- **包含字段**:
  - 手机号输入框 ← `User.phoneNumber.value`（PhoneNumber 值对象）
  - 密码输入框 ← `User.password`（提交后由后端转为 HashedPassword）
  - 确认密码输入框 ← 前端校验字段（确保与密码一致，不对应领域模型字段）
- **交互**:
  - 手机号格式校验：必须为 11 位国内手机号（来自 PhoneNumber 值对象校验规则）
  - 密码强度校验：最少 8 位（来自 User 不变量）
  - 确认密码校验：必须与密码字段一致（纯前端校验）
  - 密码和确认密码输入框右侧均有显示/隐藏切换图标（眼睛），点击切换明文/密文

### 2. 操作区

- **用途**: 提交注册请求
- **包含元素**: "注册" 主操作按钮（撑满表单宽度）
- **交互**:
  - 点击后调用 `User.register(phoneNumber, password)`
  - 成功：跳转到登录页 `/login`（不自动登录）
  - 失败：显示错误提示（手机号已注册等）

### 3. 引导区

- **用途**: 引导已有账号用户前往登录页
- **包含元素**: "已有账号？" 文案 + [立即登录] 链接
- **交互**: 点击 [立即登录] 跳转到 `/login`

## 用户操作

| 操作 | 触发方式 | 影响 |
|------|----------|------|
| 注册 | 点击 [注册] 按钮 | 调用 User.register → 成功跳转 /login，失败显示错误 |
| 跳转登录 | 点击 [立即登录] 链接 | 路由跳转到 /login |
| 切换密码可见性 | 点击密码/确认密码输入框右侧眼睛图标 | 对应输入框在明文/密文之间切换 |

## 错误状态

| 错误场景 | 提示内容 | 来源 |
|----------|----------|------|
| 手机号格式不合法 | 前端校验提示 | PhoneNumber 校验规则：11 位国内手机号 |
| 密码少于 8 位 | 前端校验提示 | User 不变量：密码最少 8 位 |
| 确认密码不一致 | 前端校验提示 | 纯前端校验 |
| 手机号已注册 | 后端返回错误 | User 不变量：手机号全局唯一 |
