## 页面信息

- **页面名称**: 注册页
- **页面类型**: 编辑页（居中表单卡片）
- **路由路径**: /register
- **所属领域**: auth

## 数据依赖

| 领域实体 | 使用方式 | 关键字段 |
|----------|----------|----------|
| User (Aggregate Root) | 注册创建目标 | phoneNumber, userName, password |
| PhoneNumber (Value Object) | 手机号输入与校验 | value（11 位国内手机号） |
| UserName (Value Object) | 用户名输入与校验 | value（英文/数字/_-，不能数字开头，3–32 位，全局唯一） |
| HashedPassword (Value Object) | 密码输入（前端提交明文，后端哈希存储） | 明文 → hash |
| Org (Aggregate Root) | 注册成功后自动创建初始 org | orgName 随机生成 |

## 区块说明

### 1. 注册表单区

- **用途**: 收集新用户注册信息
- **包含字段**:
  - 手机号输入框 ← `User.phoneNumber.value`（PhoneNumber 值对象）
  - 用户名输入框 ← `User.userName`（UserName 值对象）
  - 密码输入框 ← `User.password`（提交后由后端转为 HashedPassword）
  - 确认密码输入框 ← 前端校验字段（确保与密码一致，不对应领域模型字段）
- **交互**:
  - 手机号格式校验：必须为 11 位国内手机号
  - 用户名校验：只允许英文字母、数字、`_`、`-`；不能数字开头；长度 3–32 位；实时校验唯一性；过滤保留字（admin、root、system 等）
  - 密码强度校验：最少 8 位
  - 确认密码校验：必须与密码字段一致（纯前端校验）
  - 密码和确认密码输入框右侧均有显示/隐藏切换图标（眼睛），点击切换明文/密文

### 2. 操作区

- **用途**: 提交注册请求
- **包含元素**: "注册" 主操作按钮（撑满表单宽度）
- **交互**:
  - 点击后调用 `User.register(phoneNumber, userName, password)`
  - 成功：后端自动创建初始 org（orgName 随机），`displayName` 默认等于 `userName`，直接跳转到初始 org 工作区
  - 失败：显示错误提示

### 3. 引导区

- **用途**: 引导已有账号用户前往登录页
- **包含元素**: "已有账号？" 文案 + [立即登录] 链接
- **交互**: 点击 [立即登录] 跳转到 `/login`

## 用户操作

| 操作 | 触发方式 | 影响 |
|------|----------|------|
| 注册 | 点击 [注册] 按钮 | 调用 User.register → 成功跳转 workspace，失败显示错误 |
| 跳转登录 | 点击 [立即登录] 链接 | 路由跳转到 /login |
| 切换密码可见性 | 点击密码/确认密码输入框右侧眼睛图标 | 对应输入框在明文/密文之间切换 |

## 错误状态

| 错误场景 | 提示内容 | 来源 |
|----------|----------|------|
| 手机号格式不合法 | 前端校验提示 | PhoneNumber 校验规则：11 位国内手机号 |
| 手机号已注册 | 后端返回错误 | User 不变量：手机号全局唯一 |
| 用户名格式不合法 | 前端校验提示 | UserName 校验规则 |
| 用户名已被占用 | 实时校验提示 | UserName 全局唯一性检查 |
| 用户名是保留字 | 前端校验提示 | 静态保留字黑名单 |
| 密码少于 8 位 | 前端校验提示 | User 不变量：密码最少 8 位 |
| 确认密码不一致 | 前端校验提示 | 纯前端校验 |

## 设计决策

- **displayName 默认等于 userName**：注册时不单独收集 displayName，用 userName 兜底，用户进入产品后可随时在 Profile 页修改
- **初始 Org 自动创建**：注册成功后后端自动创建初始 org，orgName 随机生成，用户无感知，与 userName 无关联
- **userName 注册后不可修改**：userName 是登录凭证，锁死是设计决策，不提供修改入口
- **保留字用静态黑名单**：不做复杂的保留字系统，维护一份静态列表（admin、root、system、superuser、administrator 等）

## 不做什么

- 注册时不收集 displayName
- 注册时不让用户填写或选择 orgName
- 注册后不跳登录页（直接进工作区，org 已在注册时同步创建完毕）
