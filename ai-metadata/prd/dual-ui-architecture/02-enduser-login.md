## 页面信息

- **页面名称**: 用户端登录页
- **页面类型**: 表单页（公开，无需 Token）
- **路由路径**: `/end-user/[orgName]/login`
- **所属领域**: Auth（用户端）

## 数据依赖

| 领域实体 | 使用方式 | 关键字段 |
|----------|----------|----------|
| Org | 页面标题展示 Org 名称 | name, slug |
| EndUser | 登录验证 | username, password（不存储，直接传 BFF） |

## 区块说明

### 1. Org 品牌区
- **用途**: 告知 EndUser 当前登录的是哪个 Org，增强信任感
- **包含字段**: Org 名称（从 URL `orgName` 解析获取）
- **交互**: 无

### 2. 登录表单
- **用途**: 接收 username / password 并提交
- **包含字段**:
  - `username`：文本输入，必填
  - `password`：密码输入，必填，可切换明文显示
  - 登录按钮（提交中显示 loading 状态）
- **交互**: 表单提交 → `POST /api/bff/org/[orgName]/end-user/auth/login`

### 3. 错误提示区
- **用途**: 展示登录失败原因
- **包含字段**: 错误文案（条件渲染）
- **交互**: 无，仅展示

## 用户操作

| 操作 | 触发方式 | 影响 |
|------|----------|------|
| 提交登录 | 点击"登录"或按 Enter | BFF 验证 → 颁发 `aud=end-user` Token（httpOnly cookie）→ 跳转 `/end-user/[orgName]/workspace` |
| 已登录访问 | 有效 Token 访问登录页 | 自动重定向 `/end-user/[orgName]/workspace` |

## 错误状态

| 错误场景 | 提示内容 | 来源 |
|----------|----------|------|
| 用户名或密码错误 | "用户名或密码错误" | BFF 返回 401 |
| 账号已被禁用 | "账号已被管理员禁用，请联系管理员" | BFF 返回 403 |
| 该 Org 不存在 | "组织不存在，请检查链接" | BFF 返回 404 |
| 网络/服务异常 | "服务暂时不可用，请稍后重试" | BFF 返回 5xx |
| 字段为空提交 | "请填写用户名和密码" | 前端本地校验 |

## 设计决策

- 登录后**统一跳转** `/end-user/[orgName]/workspace`，无论 Project 数量多少（0/1/N 均一致）
- 不在登录页判断 Project 数量或做自动跳转逻辑，Project 选择在 Workspace 完成
- Org 名称从 URL 路径中读取，不需要额外请求（orgName 即 slug）
- Token 以 httpOnly cookie 方式存储，前端不直接读取

## 不做什么

- 不提供"忘记密码"入口（v1 不做）
- 不提供 EndUser 自助注册（账号由管理员创建）
- 不做登录后的 Project 自动跳转（统一跳 Workspace）
