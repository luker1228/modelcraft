## 页面信息

- **页面名称**: 用户端数据操作页
- **页面类型**: 列表页 + 弹窗（新增/编辑）
- **路由路径**: `/end-user/[orgName]/workspace/[projectSlug]/data`
- **所属领域**: Runtime（动态模型数据 CRUD）

## 数据依赖

| 领域实体 | 使用方式 | 关键字段 |
|----------|----------|----------|
| Model（设计时）| 侧边栏模型列表、表格列定义 | name, slug, fields |
| Field（设计时）| 表格列头、表单字段渲染规则 | name, type, jsonSchema（x-mc namespace） |
| RuntimeRecord（运行时）| 表格数据行 | 动态，由 Model 字段定义决定 |

## 区块说明

### 1. 顶部栏
- **用途**: 全局导航 + 面包屑
- **包含字段**: 面包屑（Workspace → Project 名称）、用户名、登出
- **交互**: 点击"← 返回 Workspace" → 跳转 `/end-user/[orgName]/workspace`

### 2. 侧边栏 — 模型列表
- **用途**: 在当前 Project 下的模型间切换
- **包含字段**: 模型名称列表（按 Model 的 order 排序）
- **交互**: 点击模型 → 主内容区加载对应模型的数据表格；当前选中模型高亮
- **选中逻辑**: 默认选中第一个模型；URL query param 可记录选中模型（如 `?model=xxx`）

### 3. 工具栏
- **用途**: 数据操作快捷入口 + 筛选/排序控制
- **包含字段**: "新增记录"按钮、筛选下拉、排序下拉
- **交互**:
  - 点击"新增记录" → 打开新增弹窗
  - 筛选：按字段值过滤（字段类型决定筛选器形态：文本/枚举/日期范围）
  - 排序：按可排序字段升降序

### 4. 数据表格
- **用途**: 展示当前模型的记录列表
- **包含字段**:
  - 列头由 Model 的字段定义决定（展示 field.name）
  - 每行：各字段值 + 操作列（编辑、删除）
  - 操作列固定在右侧
- **交互**:
  - 点击"编辑" → 打开编辑弹窗（预填当前行数据）
  - 点击"删除" → 弹出确认对话框 → 确认后删除

### 5. 分页控件
- **用途**: 大数据量分页
- **包含字段**: 当前页、总页数、上一页/下一页
- **交互**: 游标分页（Relay `cursor-based`），每页默认 20 条

### 6. 新增/编辑弹窗
- **用途**: 表单录入
- **包含字段**: 根据 Field 的 JSON Schema（`x-mc` namespace）动态渲染
- **字段类型 → 控件映射**:
  - `TEXT` → 文本输入框
  - `NUMBER` / `INT` → 数字输入框
  - `BOOLEAN` → 开关或单选
  - `DATE` → 日期选择器
  - `ENUM` → 下拉选择（枚举值列表）
  - `RELATION` → 关联记录选择器（搜索选择，v1 做基础版）
- **交互**:
  - 必填字段空值时阻止提交，展示错误
  - 保存成功后关闭弹窗，刷新列表
  - 取消或点击背景关闭弹窗（有未保存修改时提示确认）

## 用户操作

| 操作 | 触发方式 | 影响 |
|------|----------|------|
| 切换模型 | 点击侧边栏模型 | 主内容区重新加载对应模型数据 |
| 新增记录 | 点击"+ 新增记录" | 打开新增弹窗 |
| 保存新增 | 弹窗内点击"保存" | 调用 Runtime GraphQL createRecord → 刷新列表 |
| 编辑记录 | 点击行内"编辑" | 打开编辑弹窗（预填数据） |
| 保存编辑 | 弹窗内点击"保存" | 调用 Runtime GraphQL updateRecord → 刷新列表 |
| 删除记录 | 点击行内"删除" | 弹出确认 → 调用 deleteRecord → 刷新列表 |
| 筛选 | 展开筛选面板设置条件 | 列表重新查询 |
| 翻页 | 点击上一页/下一页 | 游标翻页查询 |
| 返回 Workspace | 点击顶部"← 返回 Workspace" | 跳转 `/end-user/[orgName]/workspace` |

## 错误状态

| 错误场景 | 提示内容 | 来源 |
|----------|----------|------|
| Token 失效 | 自动重定向登录页 | `aud` 校验失败（403） |
| 无该 Project 访问权限 | "您没有权限访问该项目" → 返回 Workspace | RBAC 校验失败 |
| 保存失败（字段校验） | 弹窗内字段级错误提示 | Runtime GraphQL 返回 InvalidInput |
| 删除失败 | "删除失败，请重试" | Runtime GraphQL 返回错误 |
| 模型列表加载失败 | 侧边栏显示"加载失败" | GraphQL 查询错误 |

## 设计决策

- 字段渲染规则完全由 JSON Schema（`x-mc` namespace）驱动，前端不硬编码字段类型判断
- 分页采用游标分页，支持大数据量场景
- 侧边栏模型列表由 Project 设计时的 Model 配置决定，不过滤（EndUser 在该 Project 下有权限则看到全部模型）
- RELATION 字段 v1 只做基础版（搜索选择），不做复杂的关联数据展示

## 不做什么

- 不提供模型结构查看/编辑功能（纯数据操作）
- 不提供批量操作（v1 不做）
- 不提供数据导出（v1 不做）
- 不提供字段级权限控制（v1 不区分字段可见性）
