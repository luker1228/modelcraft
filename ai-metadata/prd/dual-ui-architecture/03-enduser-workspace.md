## 页面信息

- **页面名称**: 用户端 Workspace 主页
- **页面类型**: 容器页（Tab 切换）
- **路由路径**: `/end-user/[orgName]/workspace`
- **所属领域**: EndUser Workspace

## 数据依赖

| 领域实体 | 使用方式 | 关键字段 |
|----------|----------|----------|
| Project | 展示 EndUser 有权访问的 Project 列表 | title, description, slug |
| EndUserRoleUser | 决定哪些 Project 可以展示 | project_slug（聚合后去重） |

## 区块说明

### 1. 顶部栏
- **用途**: 全局导航，展示当前身份上下文
- **包含字段**: Org 名称、当前 EndUser 用户名、登出按钮
- **交互**: 点击"登出" → 清除 httpOnly cookie → 跳转 `/end-user/[orgName]/login`

### 2. Tab 导航
- **用途**: 功能区切换
- **包含字段**:
  - Tab 1：`Projects`（当前版本主要功能）
  - Tab 2：待定（预留，后期可扩展为 API Key 管理等）
- **交互**: 点击 Tab 切换内容区

### 3. Projects Tab — Project 卡片列表
- **用途**: 展示 EndUser 有权访问的 Project，供选择进入
- **包含字段**: Project title、description（可选）、进入按钮
- **交互**: 点击卡片或"进入 →" → 跳转 `/end-user/[orgName]/workspace/[projectSlug]/data`
- **布局**: 卡片网格，响应式（桌面 3 列，平板 2 列，移动 1 列）

### 4. 空状态区（条件显示）
- **用途**: 无可访问 Project 时引导 EndUser
- **包含字段**: 提示文案："您暂无项目访问权限，请联系管理员授权"
- **交互**: 无（v1 不提供自助申请入口）

## 用户操作

| 操作 | 触发方式 | 影响 |
|------|----------|------|
| 进入 Project | 点击 Project 卡片 / "进入 →" | 跳转 `/end-user/[orgName]/workspace/[projectSlug]/data` |
| 切换 Tab | 点击 Tab 标签 | 切换内容区（Tab 2 内容待定） |
| 登出 | 点击"登出" | 清除 cookie → 跳转登录页 |
| 刷新 | 浏览器刷新 | 重新请求 Project 列表 |

## 错误状态

| 错误场景 | 提示内容 | 来源 |
|----------|----------|------|
| Token 失效 | 自动重定向登录页 | 中间件 `aud` 校验失败（403） |
| Project 列表加载失败 | "加载失败，请刷新重试" | GraphQL 查询错误 |
| 无可访问 Project | 空状态提示 | Project 列表为空 |

## 设计决策

- Workspace 是登录后的**唯一落地页**，不根据 Project 数量做自动跳转（0/1/N 都到这里）
- Tab 2 预留占位，不在 v1 实现，但 Tab 组件结构要支持扩展
- Project 列表通过 GraphQL 查询获取（后端按 `end_user_role_users` 聚合去重），前端不做二次权限过滤

## 不做什么

- 不在此页面提供模型/数据操作功能
- 不显示 Project 内的模型列表（在数据页侧边栏展示）
- Tab 2 不实现（仅预留结构）
- 不提供 Project 搜索（v1，Project 数量预计较少）
