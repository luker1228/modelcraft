# 🎯 设计理念

> **优先级: 最高** - 本目录内容是项目的核心原则，不可动摇。任何与此冲突的实现都应以设计为准。

## 概述

设计理念定义了项目的核心价值观、领域模型和业务规则。这些原则指导所有后续的技术决策。

## 🔑 核心原则摘要

### 1. 领域驱动设计 (DDD)

- 业务逻辑集中在 Domain 层
- 领域模型是系统的核心
- 使用通用语言 (Ubiquitous Language)

### 2. 清洁架构

- 依赖方向：外层依赖内层
- Domain 层不依赖任何外部层
- 通过接口实现依赖反转

### 3. 关注点分离

- 每层有明确的职责
- 业务逻辑与技术实现分离
- 接口与实现分离

## ⚠️ 不可违背的约束

1. **Domain 层纯净性** - Domain 层不能有任何外部依赖
2. **业务逻辑位置** - 所有业务规则必须在 Domain 层实现
3. **数据验证** - 领域对象在创建时必须验证有效性
4. **实体标识** - 实体通过 ID 标识，值对象通过属性标识
