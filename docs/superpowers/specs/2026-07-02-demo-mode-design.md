# Demo Mode（匿名体验模式）设计文档

> 日期：2026-07-02
> 目标：允许访客无需注册登录即可体验 ModelCraft，以只读 + 临时 API Key 方式操作预置的 demo 数据

---

## 背景

ModelCraft 是面向开发团队的低代码数据模型管理平台。为了让潜在用户（技术开发者）快速体验核心价值——连接数据库、查看模型、调用运行态 GraphQL API——需要一个无需登录的匿名体验入口。

---

## 整体架构

```
服务启动 seed → 创建 demo org + Guest 角色 + 权限配置
Owner 手动操作 → 登录 demo org，连接数据库，定义模型（一次性配置）
                         ↓
登录页「匿名体验」按钮 → POST /auth/demo → 返回 Guest JWT
                         ↓
访客以 Guest 身份进入 demo org，浏览模型，创建 10min 临时 API Key，调用运行态 GraphQL
```

---

## 一、系统初始化（Seed）

服务启动时幂等执行以下操作：

- 创建固定 `demo` org，owner 账号密码通过环境变量配置（由平台管理员掌握）
- 在 demo org 下创建 `Guest` 系统角色
- Guest 角色权限：
  - 只读：查看模型、字段、关联关系
  - 允许创建临时 API Key，TTL 上限 10min
  - 禁止：创建/修改/删除模型、邀请用户、修改组织配置等所有写操作

Seed 完成后，平台管理员用 owner 账号登录，手动连接数据库并定义模型（一次性操作，之后固定不动）。

---

## 二、认证与会话

新增公开接口：

```
POST /auth/demo
无需请求参数，无需鉴权
返回：Guest JWT（org=demo，role=Guest）
```

- JWT 有效期建议 2 小时（访客短会话，无需长期保持）
- 前端拿到 JWT 后走现有会话流程，自动进入 demo org 上下文
- Guest 会话与普通用户会话代码路径完全一致，区别仅在角色权限

---

## 三、API Key 临时创建

复用现有 API Key 创建流程，在权限层增加以下约束：

- Guest 角色创建 API Key 时，TTL 强制上限 30min（后端校验）
- Key 权限范围：仅限 demo org 下的运行态 GraphQL 查询
- 30min 后自动失效，访客需重新创建

后端改动极小：在 API Key 创建逻辑中，检测当前角色为 Guest 时，强制 TTL ≤ 10min。

---

## 四、前端改动

改动点仅两处：

1. **登录页**：新增「匿名体验」按钮，点击后调用 `POST /auth/demo`，拿到 JWT 后跳转进入 demo org
2. **Demo Banner**：进入 demo org 后顶部显示 Banner，提示「演示模式 · 数据仅供体验」

不在前端做写操作入口的屏蔽，后端权限校验是唯一防线。

---

## 五、登录方式变更

现有手机号登录**全量替换**为用户名 + 密码登录：

- 所有用户统一使用用户名 + 密码认证
- 手机号登录接口废弃，无存量数据，直接破坏性替换，无需迁移
- Seed 时直接写入 demo org owner 的用户名和密码（通过环境变量配置），初始化链路完整

---

## 六、改动范围总结

| 模块 | 改动内容 |
|------|---------|
| 后端 - Auth | 手机号登录替换为用户名 + 密码登录 |
| 后端 - Seed | 新增 demo org + Guest 角色 + owner 账号初始化逻辑 |
| 后端 - Auth | 新增 `POST /auth/demo` 公开接口 |
| 后端 - RBAC | 新增 Guest 角色权限定义 |
| 后端 - API Key | Guest 角色创建 API Key 时 TTL 上限校验 |
| 前端 - 登录页 | 手机号替换为用户名 + 密码输入，新增「匿名体验」按钮 |
| 前端 - Layout | 新增 Demo Banner 组件 |

---

## 六、不在本次范围内

- 访客行为统计/埋点
- Demo 数据的自动重置（数据由 owner 手动维护）
- 多 demo org 支持
