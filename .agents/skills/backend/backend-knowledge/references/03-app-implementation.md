# 03 - App 开发指引

## 职责

`app` 层是业务流程编排层：

- 组织一个完整用例（Use Case）
- 协调 domain 与 repo
- 管理事务边界
- 控制调用顺序和权限

## 设计原则

1. **面向用例**：一个方法对应一个业务动作
2. **编排而不臃肿**：规则放 domain，持久化放 repo
3. **可观测**：关键节点打日志与错误上下文

## 典型流程

1. 校验业务前置条件
2. 调用 domain 规则计算
3. 调用 repo 落库或查询
4. 组装输出对象返回 interface

## 事务建议

- 涉及多次写操作时由 app 控制事务
- repo 只关注 SQL/存储本身

## 常见反模式

- app 直接拼 SQL
- app 内重复实现领域规则
- 把 app 做成“巨型服务”不拆分用例
