# 05 - Repo 开发指引

## 职责

`repo` 层负责数据访问：

- SQL 查询与写入
- 数据模型与领域对象转换
- 事务内的数据操作
- 屏蔽存储实现细节

## 设计原则

1. **面向接口**：由 app 依赖 repo 接口，不依赖具体实现
2. **单一职责**：专注数据读写，不承载业务判断
3. **可测试**：支持 mock 与集成测试

## 开发步骤

1. 在 app/domain 先定义 repo 接口契约
2. 在 repo 层实现查询/写入逻辑
3. 做数据对象映射（DB Model ↔ Domain）
4. 处理存储层错误并附带上下文

## SQL 组织建议

- 按聚合根或业务模块拆分查询文件
- 复杂查询封装成明确命名方法
- 避免在 app 层拼接 SQL

## 常见反模式

- repo 返回过于底层的数据库细节给上层
- 在 repo 层做业务流程编排
- 把多个无关聚合混在同一个 repo 实现
