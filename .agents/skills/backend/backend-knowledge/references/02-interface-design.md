# 02 - Interface 开发指引

## 职责

`interface` 层负责系统边界：

- GraphQL/HTTP 输入解析
- 请求参数校验
- DTO 与应用层模型映射
- 错误到响应格式的转换

## 设计原则

1. **瘦接口**：只做边界处理，不承载业务流程
2. **明确契约**：输入输出结构稳定、字段语义清晰
3. **失败前置**：参数校验尽早返回

## 开发步骤

1. 定义输入输出结构（Input/Output DTO）
2. 完成参数校验与默认值处理
3. 调用 app 层方法
4. 统一封装返回与错误映射

## 校验建议

- 格式校验：长度、枚举、必填
- 业务校验：交由 app/domain

## 常见反模式

- 在 resolver/handler 写复杂业务逻辑
- 直接操作数据库（绕过 app/repo）
- 将内部错误细节直接暴露给客户端
