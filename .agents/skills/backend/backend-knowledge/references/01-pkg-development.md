# 01 - Pkg 开发指引

## 职责

`pkg` 层是公共能力层，提供可复用基础设施：

- 配置读取
- 日志组件
- 中间件
- 通用工具函数
- 外部服务客户端封装

## 设计原则

1. **通用性优先**：不包含具体业务含义
2. **接口稳定**：向上层暴露简单、明确的 API
3. **低耦合**：不依赖 app/domain/interface
4. **可配置**：通过构造参数注入配置

## 推荐结构

```text
pkg/
  logger/
    logger.go
  errors/
    codes.go
    wrap.go
  config/
    config.go
  utils/
    time.go
```

## 开发步骤

1. 先定义对外接口（函数或结构）
2. 再实现内部细节
3. 添加最小可用测试（边界 + 错误路径）
4. 提供示例调用

## 常见反模式

- 在 pkg 中写入业务规则
- 在 pkg 中直接访问 repo/domain
- 导出过多细节（暴露内部结构）
