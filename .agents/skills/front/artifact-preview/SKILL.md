# Artifact Preview

Start a local HTTP server to preview HTML artifacts generated by web-artifacts-builder or other tools.

## Quick Start

After generating an artifact with web-artifacts-builder, use this skill to preview it:

```bash
python3 scripts/serve_artifact.py /path/to/artifact-directory
```

The script will:
1. Kill any process using port 9999
2. Start HTTP server on port 9999
3. Display access URL: `http://localhost:9999/bundle.html`

## Usage Workflow

### 1. Locate the artifact directory

Web-artifacts-builder typically generates artifacts in:
```
.agents/skills/web-artifacts-builder/<project-name>/
```

The directory contains:
- `bundle.html` - The main artifact file
- Other assets (if any)

### 2. Start the preview server

Run the serve script with the artifact directory path:

```bash
python3 scripts/serve_artifact.py .agents/skills/web-artifacts-builder/editor-sidebar-redesign
```

### 3. Access the artifact

Open browser to `http://localhost:9999/bundle.html`

### 4. Stop the server

Press `Ctrl+C` to stop the server when done.

## Port Conflict Handling

The script automatically handles port 9999 conflicts:
- Detects processes using port 9999
- Terminates conflicting processes
- Starts the server cleanly

No manual intervention needed for port conflicts.

## Common Patterns

**After building an artifact:**
```bash
# User: "Build me a dashboard"
# web-artifacts-builder creates: .agents/skills/web-artifacts-builder/dashboard/bundle.html
# Then use this skill:
python3 scripts/serve_artifact.py .agents/skills/web-artifacts-builder/dashboard
```

**Testing multiple artifacts:**
```bash
# Stop current server (Ctrl+C)
# Start new server for different artifact
python3 scripts/serve_artifact.py .agents/skills/web-artifacts-builder/another-project
```

## Script Reference

The skill includes `scripts/serve_artifact.py` which:
- Accepts artifact directory path as argument
- Validates directory and checks for bundle.html
- Handles port conflicts automatically using `lsof` or `netstat`
- Starts Python's built-in HTTP server on port 9999
- Provides clear status messages and access URL
