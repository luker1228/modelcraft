# 代码风格规范

编写 Go 代码时，必须遵守命名约定、错误处理、日志、协程等代码风格规范。

Refer to @ai-metadata/backend/development/code-style.md for naming conventions, error handling patterns, logging usage, transaction best practices, and goroutine safety guidelines.

## 触发场景

- 编写任何 Go 代码时
- 创建新的函数、类型、接口时
- 处理错误或打印日志时
- 使用事务或启动协程时
