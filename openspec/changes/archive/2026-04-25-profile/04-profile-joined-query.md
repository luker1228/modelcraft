## 页面信息

- **页面名称**: 联合查询
- **页面类型**: 查询页
- **路由路径**: /profile
- **所属领域**: 用户资料 / BFF 聚合

## 数据依赖

| 领域实体 | 使用方式 | 关键字段 |
|----------|----------|----------|
| User | 返回账号基础信息 | id, phone, user_name, status |
| Profile | 返回资料扩展信息 | user_id, nickname, avatar_url(mock), bio |

## 区块说明

### 1. 查询入口区块
- **用途**: 提供单次请求拉取完整用户信息。
- **包含字段**: userId / 当前登录态。
- **交互**: 前端页面加载时触发。

### 2. 聚合读取区块
- **用途**: 并行/串行读取 user 与 profile。
- **包含字段**: user.*、profile.*。
- **交互**: 后端聚合成统一 DTO 返回调用方。

### 3. 缺失兜底区块
- **用途**: 处理 profile 缺失或异常场景。
- **包含字段**: nickname/avatar(mock)/bio 默认值。
- **交互**: 避免前端因空对象崩溃。

## 用户操作

| 操作 | 触发方式 | 影响 |
|------|----------|------|
| 打开个人信息页 | 页面初始化 | 一次请求获取完整信息 |
| 刷新资料 | 主动刷新或编辑成功后回调 | 获取最新 user/profile 数据 |

## 错误状态

| 错误场景 | 提示内容 | 来源 |
|----------|----------|------|
| user 不存在 | `user not found` | User 查询 |
| profile 缺失 | `profile missing, fallback applied` | 数据一致性 |
| 聚合超时 | `failed to load user profile` | 聚合接口超时 |

## 设计决策

- 对外暴露聚合查询，减少前端/BFF 多次请求与拼装复杂度。
- profile 缺失时返回可渲染兜底值，并记录告警，保障可用性。

## 不做什么

- 不在本次提供批量联合查询（list 场景）。
- 不在本次引入分页版本的单用户详情查询。
