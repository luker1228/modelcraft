# Profile 分表（User / Profile Separation）

## 是什么问题
当前用户主表同时承载账号标识与资料信息，导致模型职责混杂，后续扩展资料字段（如头像、简介、偏好设置）时会增加主表复杂度。需要将账号主体与资料信息拆分为 `user` 与 `profile` 两张表。

## 目标用户
- 平台终端用户：注册、维护个人资料
- 后端/前端研发团队：以更清晰的领域边界实现用户域能力

## 用户故事
- 作为新注册用户，我希望使用手机号完成注册并自动创建我的 Profile，以便后续可以直接补全资料。
- 作为已登录用户，我希望编辑我的 Profile（昵称、头像、简介等），以便展示更完整的个人信息。
- 作为调用方（前端/BFF），我希望一次查询能拿到 User + Profile，以便渲染完整用户信息页。

## 核心功能
1. **数据模型拆分**：新增 `profile` 表，`user` 表仅保留账号主体字段（至少含 `id`、`phone`、`userName`）；`profile` 通过外键关联 `user`。
2. **注册联动创建**：用户使用手机号注册成功后，系统自动创建对应 `profile` 记录，并写入默认 `nickname`。
3. **Profile 编辑能力**：提供更新 Profile 的接口（至少覆盖昵称、头像、简介）；头像当前采用 mock 资源。
4. **联合查询能力**：提供 User + Profile 查询接口，支持前端一次获取完整用户信息。

## 不做什么
- 本次不引入社交关系（关注/粉丝）模型。
- 本次不做多 profile（一个 user 对应多个 profile）设计。
- 本次不做复杂隐私可见性策略（如字段级可见范围控制）。
- 本次不接入真实头像上传/存储服务（头像仅使用 mock 资源）。

## 待确认
